# ADR-004: Uso del patrón Component

## Estado
Propuesto

## Contexto
El frontend requiere una estructura organizada y reutilizable
para manejar diferentes vistas como productos y carrito.

## Decisión
Se utilizará el patrón Component mediante la arquitectura de Angular,
dividiendo la aplicación en componentes independientes.

## Alternativas consideradas
1. Código monolítico en un solo componente
2. División en múltiples componentes (Component)

## Consecuencias
Positivas:
- Reutilización de código
- Mejor organización
- Escalabilidad

## Negativas:
- Mayor cantidad de archivos
